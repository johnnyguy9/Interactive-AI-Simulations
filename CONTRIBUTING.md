# Contributing

Thanks for taking a look. This is a personal / portfolio project, but issues and pull requests are welcome.

## Reporting bugs

1. Search existing issues first.
2. Open a new issue with:
   - A clear summary.
   - Steps to reproduce, including OS / runtime versions.
   - Expected vs. actual behavior.
   - Logs, screenshots, or a minimal reproduction if relevant.

## Proposing changes

1. Open an issue describing the change before sending a large PR. Small fixes (typos, dead links, obvious bugs) can skip this.
2. Fork the repo and create a feature branch from `main`.
3. Make focused commits with clear messages.
4. Run any linting / tests defined in this repo before pushing.
5. Open a pull request against `main` and link the issue it resolves.

## Code style

- Match the existing style of the file you are editing.
- Keep changes scoped — one concern per PR is easier to review and ship.
- Add or update documentation when behavior changes.
